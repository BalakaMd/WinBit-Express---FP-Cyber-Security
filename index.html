Verifiable Security Requirements

1. Multi-Factor Authentication (MFA)

Given: User has an account in the system

When: User attempts to log in

Then: System requires second factor (TOTP/Push/WebAuthn)

2. Data Encryption

Given: Data is stored in CRM or transmitted via API

When: Access attempt or traffic analysis occurs

Then: Data only accessible in encrypted form (AES-256 at rest, TLS 1.3 in transit)

3. Centralized Audit & Logging

Given: Employee or client performs action (create, modify, delete)

When: Repudiation attempt or incident occurs

Then: SIEM contains complete event log with immutable records (WORM)

4. DoS/DDoS Protection

Given: Web/mobile app receives increased load

When: Request volume exceeds acceptable thresholds

Then: WAF/CDN blocks suspicious traffic, system remains available

5. Role Separation & Least Privilege

Given: Employee has business role (Accountant, CCO, Sysadmin)

When: Attempts action beyond their authority

Then: Access denied, attempt logged

ATT&CK ‚Üí NIST Control Mapping

1. MFA Implementation

ATT&CK: T1078 (Valid Accounts), T1566 (Phishing)

NIST: IA-2 (MFA), AC-2 (Account Management)

  • Keycloak + WebAuthn (FIDO2/U2F) for employees
  • Auth0 / Azure AD / Okta MFA for clients
  • Hardware tokens (YubiKey) for CISO/admins

2. Encryption Strategy

ATT&CK: T1041 (Exfiltration Over C2)

NIST: SC-13 (Cryptographic Protection), SC-8 (Transmission Confidentiality)

  • AES-256 + PostgreSQL TDE / MongoDB Encrypted Storage
  • TLS 1.3 (LetsEncrypt/Cloudflare certificates)
  • HashiCorp Vault / AWS KMS for key management

3. Centralized Logging

ATT&CK: T1070 (Indicator Removal)

NIST: AU-2 (Audit Events), AU-6 (Review), SI-4 (Monitoring)

  • Elasticsearch + Kibana SIEM
  • Filebeat/Logstash ‚Üí WORM storage (immutable S3/MinIO)
  • OSSEC/Wazuh for server logs

ATT&CK ‚Üí NIST Mapping (Continued)

4. DDoS Protection

ATT&CK: T1498 (Network DoS)

NIST: SC-5 (DoS Protection), CP-10 (System Recovery)

  • Cloudflare WAF + CDN (rate limiting, bot filtering)
  • Hetzner DDoS protection + backup channels
  • Runbooks in PagerDuty/Ansible for recovery

5. Role Separation & Access Control

ATT&CK: T1078 (Valid Accounts)

NIST: AC-2 (Account Management), AC-3 (Access Enforcement)

  • RBAC in CRM (role-based models)
  • IAM policies (AWS/GCP IAM or Keycloak ABAC)
  • Quarterly access reviews (AuditBoard, SailPoint)

Incident Response & Monitoring

1. Preparation

Actions:
  • Create playbooks for key scenarios (phishing, data breach, DoS, account compromise)
  • Define Incident Response Team (CISO, SOC Analyst, Sysadmin, Compliance, DPO)
  • Conduct bi-annual training (table-top exercises)
Tools: Jira/Confluence, Slack/Teams IR channels, PagerDuty
Metrics: % employees trained, playbook currency

2. Detection & Analysis

Actions:
  • Monitor SIEM for suspicious events: T1078, T1498, T1070
  • UEBA for CRM anomalies
  • Automated SOC alerts for critical events (failed MFA, mass API calls)
Metrics: MTTD (Mean Time to Detect), % events investigated within SLA

3. Containment, Eradication, Recovery

Actions:
  • Isolate compromised accounts or servers
  • Restore from backups (Hetzner snapshots, WORM logs)
  • Activate DDoS mitigation (Cloudflare/Hetzner)
  • Notify clients and regulators (GDPR 72h, DORA reporting)
Tools: ESET endpoint protection, Ansible, Cloudflare API
Metrics: MTTR (Mean Time to Recover), RPO/RTO compliance

Continuous Monitoring Framework

4. Post-Incident & Lessons Learned

  • Post-mortem meetings with all stakeholders
  • Update controls (NIST, ISO, GDPR/DORA compliance)
  • Document incidents in knowledge base
Tools: Confluence templates, Jira backlog

5. Continuous Risk Monitoring

Daily: SOC monitoring of logs and alerts
Quarterly: Vulnerability scans (Nessus, OpenVAS)
Annually: Penetration testing and ISO27001 audit
Ongoing: CVE, NVD, CISA KEV analysis
